Key Business Rules Clarified

- TL = unregistered seed.
- Correct operational flow:
  Inward -> Purchases/raw receipt -> Grading -> Packing -> Inventory -> New Invoice.
- Raw inward stock should not be directly available in New Invoice.

- RATE will stay in Packing for now.

  - RATE is saved to packing_sessions and copied to seed_lots.
  - Inventory and New Invoice use that seed_lot rate.

Formula Notes
- Inward formulas match the workbook notes:
  - Net Wt = First Wt - Second Wt
  - Bag Wgt = Bags * Size
  - Wgt-Bag = (Net Wt - Bag Wgt) / 100
  - BOUNCING = ROUNDUP((Net Wt / 90) * 0.003, 2)
  - NET WEIGHT = Wgt-Bag - BOUNCING
  - AMOUNT = RATE * NET WEIGHT
- Current code uses Math.max(First Wt - Second Wt, 0), so negative Net Wt is prevented.
- Grading formulas:
  - GRADED QUANTITY = GRADED BAGS * GRADED BAG SIZE / 100
  - Total Graded = GRADED QUANTITY + GRADED LOOSE
  - UNDERSIZE % = UNDERSIZE / WEIGHT RECPT * 100
  - Final Graded = Total Graded + UNDERSIZE
  - Current app Difference = WEIGHT RECPT - Final Graded
- Packing formulas:
  - Already packed quantity = sum of previous packing_sessions.total_packed for supplier + variety
  - Available graded quantity = Total graded quantity - Already packed quantity
  - Packing Quantity = NO OF BAGS * PACKING BAG SIZE / 100
  - Total packed = Packing Quantity + PACKING LOOSE + GHAMSI
  - Remaining after packing = Available graded quantity - Total packed
- Purchases Database-style report:
  - Remaining Gradings = Wgt-BagS - Total Graded QT - U/s
  - Remaining packings = Total Graded QT - Packed Qty

Supabase SQL/Migration Notes
- Live Supabase may need these columns on packing_sessions:
  alter table packing_sessions
  add column if not exists seed_type text not null default 'VS' check (seed_type in ('VS', 'TL'));

  alter table packing_sessions
  add column if not exists rate numeric(12,2) not null default 0;

  alter table packing_sessions
  add column if not exists seed_lot_id uuid references seed_lots(id) on delete set null;

  notify pgrst, 'reload schema';
- seed_lot_availability view must expose source_state.
- Correct view replacement appends source_state at the end to avoid PostgreSQL column-order error:
  create or replace view seed_lot_availability as
  select
    sl.company_id,
    sl.id as seed_lot_id,
    p.crop,
    p.variety_name,
    p.display_name,
    sl.lot_number,
    sl.seed_class,
    sl.packing_kg,
    sl.rate,
    sl.opening_bags,
    sl.current_bags,
    sl.hold_bags,
    greatest(sl.current_bags - sl.hold_bags, 0) as available_bags,
    (greatest(sl.current_bags - sl.hold_bags, 0) * sl.packing_kg) / 100 as available_quintal,
    ((greatest(sl.current_bags - sl.hold_bags, 0) * sl.packing_kg) / 100) * sl.rate as available_value,
    case
      when greatest(sl.current_bags - sl.hold_bags, 0) <= 0 then 'sold_out'
      when greatest(sl.current_bags - sl.hold_bags, 0) <= 10 then 'low_stock'
      else 'healthy'
    end as stock_status,
    sl.source_state
  from seed_lots sl
  join products p on p.id = sl.product_id;

  notify pgrst, 'reload schema';

Known Issue / Caution
- Packing save currently creates/updates seed_lots before inserting packing_sessions.
- If packing_sessions insert fails after the seed_lot write, packed stock can be created/incremented even though the UI shows an error.
- This should be fixed later by moving Packing save into a database RPC/transaction.
- If the user retried failed packing saves, check for inflated current_bags in seed_lots.
